# CURRENT_STATE: "Abstractor" Android Application

Welcome to the **Abstractor** project. This file serves as the definitive onboarding guide and architectural blueprint for Gemini LLM chats. It details the complete system design, technical stack, file architecture, and precise implementation rules.

---

## 🚀 Overview & Mission
**Abstractor** is a professional-grade Jetpack Compose Android application designed for knowledge workers. It extracts, analyzes, and summarizes digital content (web pages, documents, media) into highly structured, actionable intelligence using Google's Gemini models.

Key highlights:
- **No Filler Policy**: Synthesizes source material without generic summaries, abbreviations, or over-simplifications.
- **Deep Source Analysis**: Processes the *entire* document/transcript instead of partial text windows.
- **Dynamic Context Scanning**: Automatically adjusts analysis depth based on the style and caliber of the input source.

---

## 🛠️ Tech Stack & Architecture
The system is built on modern, lightweight, and robust Android paradigms:
- **Language**: Kotlin (100% Type-Safe)
- **UI Toolkit**: Jetpack Compose (Material Design 3, Fluid Edge-to-Edge)
- **Concurrency**: Kotlin Coroutines & Cold Flows
- **API Communication**: Retrofit 2 & OkHttp (JSON-based REST API)
- **Architecture**: Clean MVVM (Model-View-ViewModel)
- **Model Provider**: Google Gemini API with JSON response schema enforcement and advanced fallback parsing.

## 📂 Codebase Landscape & Core Components

### 1. `MainActivity.kt` (UI Layer & Entry point)
The primary controller managing view rendering and state changes:
- **Startseite (Idle)**: Prompts for URLs. Features one-click clipboard fetching (`LocalClipboardManager`), persistent input values (so the user can re-analyze the same URL under different focus modes), and the "Cockpit" (an intuitive grid of 8 direct action buttons).
- **Detailseite (Analysis & Success)**: 
  - Displays dynamic title states based on the active `AnalysisType`.
  - Displays a premium **Advanced Metadata Card** containing:
    - **Document Title** (LLM-extracted Title)
    - **Original Source URL**: (Clean formatting with monospace typography)
    - **Author / Publisher (Owner)**: (If extracted, using the newly engineered `owner` field).
  - Skips descriptions entirely in compressed modes (like `3 Kernaspekte`).
  - Implements customized section titles (e.g., `"ZWEIFELHAFTE INFORMATIONEN"` for Fehlformations-Radar).
  - Implements beautiful, individually boxed numbered list indicators for `TOP_3_KERNAUSSAGEN` and `RISIKO_ANALYSE`.

### 2. `GeminiNetwork.kt` (Integration & Prompt Tuning)
The analytical powerhouse of the application that manages communications, schema definitions, and prompt tuning:
- **Global Constraints**: Injects unyielding instruction wrappers guaranteeing complete context analysis with zero omissions or superficial skimming.
- **Strict Response Schema**: Translates raw results into `AbstractorSummary` instances.
- **Robust Regex fallback Parser (`parseSummaryRobustly`)**: Guarantees recovery of partially malformed JSON strings, ensuring the application remains resilient under unstable network payloads or varying LLM behaviors.

### 3. `GeminiModels.kt` (Domain & Types)
Defines structured classes:
- `AnalysisType`: Enum representing the 8 focus analysis modes.
- `UiState`: Sealed class representing execution flows (`Idle`, `Loading`, `Success`, `Error`).
- `AbstractorSummary`: Model holding `title`, `originalUrl`, `shortDescription`, `keyTakeaways`, and the new `owner` field.

## 🎯 Fine-Tuned Analysis Modes

Here is the exact state of the 8 focus modes after our comprehensive architectural and prompt refactoring:

### 1. STANDARD_WEBSEITE (Standard-Webseite)
- **Focus**: Thorough, objective, and deeply professional deutschen summary.
- **Takeaways**: Un-numbered high-density bullet points.

### 2. MULTIMEDIA (Multimedia & YouTube)
- **Focus**: Extracts core topics, timestamps, facts, and speakers from audio or YouTube transcripts.

### 3. DOKUMENTE (Fachdokumente)
- **Focus**: Rigorous academic dissection of research papers, PDFs, and legal articles. Focuses heavily on methodologies, evidence, and figures.

### 4. TOP_3_KERNAUSSAGEN (3 Kernpunkte) - *Refactored, Validated & Hardened*
- **Focus**: Zero-fluff representation of the absolute pillars.
- **Constraints**: 
  - Strictly forbids Fließtext-Zusammenfassungen or generic descriptions.
  - Implements **strict content validation**: if the extracted text or YouTube transcript is under 500 characters or fails entirely, the app rejects summarizing using URL-only or metadata contexts to avoid false/hallucinated takeaways from LLM world-knowledge.
  - If direct scraping fails, falls back strictly to Google Search Grounding. If search grounding also fails, displays a helpful, descriptive "Inhalt nicht auslesbar" error summary.
  - Removed all LLM prefix numbering requirements from prompts and parsing logic to preserve structural parsing integrity while elegantly displaying clean, custom bullet indexes in the UI.

### 5. AKTUALITAETS_CHECK (Prüfe Aktualität) - *Refactored & Hardened*
- **Focus**: Two-dimensional evaluation of timeliness:
  - **Dimension A**: Physical publication date (timestamp of release).
  - **Dimension B**: Actual in-use relevancy (whether the text's contents/claims are still factually correct or have been superseded by newer studies/events).
- **UI Presentation**: Tabular/bullet listings showing the distinct dimension evaluations under an `"AKTUALITÄTS-DETAILS (ZWEIDIMENSIONAL)"` heading.

### 6. FEHLINFORMATIONS_RADAR (Fehlinformations-Radar) - *Refactored & Hardened*
- **Focus**: Mediacompetency sweep checking for logical fallacies, manipulations, clickbait patterns, and unverified data claims.
- **UI Presentation**: Expressed under the custom heading **"Zweihafte Informationen"** (formerly labeled as takeaways).

### 7. RISIKO_ANALYSE (Identifiziere Risiken) - *Refactored & Hardened*
- **Focus**: Strategic risk evaluation identifying economic, technical, or regulatory blindspots.
- **Structure**:
  - `shortDescription`: A robust, professional two-sentence summary describing the general risk profile.
  - `keyTakeaways`: A clean, numbered list detailing specific concrete risks arising from the document's content.

### 8. BUSINESS_INKUBATOR (Business-Inkubator)
- **Focus**: Explores raw text for prospective SaaS ideas, business opportunities, or systematic market inefficiencies.
